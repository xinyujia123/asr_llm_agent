# 护理文书生成测试集

本目录用于评测模型在护理场景下把护士口述、ASR 转写或简要病情记录生成规范护理文书的能力。测试样例覆盖入院评估、术后观察、输血反应、低血糖、突发胸痛、跌倒、压力性损伤、PICC 维护、留置尿管感染风险、产后出血风险、化疗药物外渗和出院健康教育等场景。

## 文件说明

- `dataset.jsonl`：护理文书生成测试集，每行一个 JSON 样例。
- `prompt.md`：建议用于该场景的系统提示词，要求模型只输出 JSON。
- `README.md`：场景说明、评判重点和字段说明。

## 数据集字段

- `id`：样例唯一编号。
- `scenario`：样例对应的护理文书场景。
- `input`：护士口述、ASR 转写或病情摘要。
- `expected_output`：优秀参考答案，采用结构化护理文书对象，包含文书类型、患者信息、记录时间、护理评估、护理问题、护理措施、效果评价、风险标记和后续观察/交接重点。

## 评判重点

1. JSON 合法性：输出必须是单个合法 JSON 对象，不能包含 Markdown、解释文字或多余前后缀。
2. 信息忠实度：不得编造输入中没有的医嘱、剂量、检查结果、诊断或处置结局；能正确处理口述纠正和不确定信息。
3. 护理专业性：评估、护理问题、措施、效果评价和交接重点符合护理文书习惯，措辞客观、准确、可追溯。
4. 风险识别：能识别并突出低氧、胸痛、输血反应、低血糖、跌倒、压力性损伤、管路感染、产后出血、化疗外渗等高风险点。
5. 处置闭环：高风险事件应体现发现、评估、报告医生/专科人员、执行医嘱或护理措施、复评效果、持续观察和交接。
6. 结构完整性：关键字段应尽量完整，缺失或不确定信息应放入对应字段，而不是擅自补全。
