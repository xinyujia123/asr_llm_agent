# 出院宣教测试集

## 场景说明

本目录用于评测护理大模型在“出院宣教”场景中的生成能力。输入为患者出院前的简要病情、治疗经过、带药、管路/伤口情况和患者问题；模型应输出结构化、可执行、患者友好的中文出院宣教。

数据覆盖普外科术后、慢阻肺、心力衰竭、剖宫产、脑卒中、PICC带管、髋关节置换、糖尿病足、慢性肾脏病、儿童哮喘、甲状腺术后、心梗PCI术后等类型，包含老年、孕产、慢病、术后、伤口及管路护理等重点人群和场景。

## 评判重点

- 是否只基于输入信息生成宣教，避免编造剂量、检查结果、诊断或复诊日期。
- 是否覆盖用药、复诊、伤口/管路、饮食运动、警示症状、居家照护和患者友好表达。
- 是否符合护理专业规范，能区分常规观察、提前复诊和急诊处理。
- 是否对慢病管理、术后恢复、管路安全、跌倒预防、感染预防、康复训练等给出可执行建议。
- 是否语言清楚温和，患者和家属能理解并照做。
- 是否输出合法 JSON，字段完整，数组内容具体。

## 数据集字段

`dataset.jsonl` 每行一个 JSON 对象，主要字段如下：

- `id`：样例编号。
- `scenario`：固定为“出院宣教”。
- `input`：患者出院信息和患者/家属关注问题。
- `expected_output`：优秀参考结果，包含：
  - `summary`：护理重点概括。
  - `medication`：用药宣教。
  - `follow_up`：复诊安排和提前就诊提示。
  - `wound_tube_care`：伤口、敷料、管路或无管路时的观察说明。
  - `diet_activity`：饮食、饮水、运动和康复建议。
  - `warning_symptoms`：警示症状。
  - `home_care`：居家照护和监测。
  - `patient_friendly_expression`：面向患者/家属的通俗说明。

`prompt.md` 为该场景推荐系统提示词，要求模型只输出 JSON，并包含出院宣教必要字段。
