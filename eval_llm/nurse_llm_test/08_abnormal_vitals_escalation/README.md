# 异常生命体征升级建议测试集

## 场景说明

本目录用于评测护理大模型在“异常生命体征升级建议”场景中的生成能力。输入为病区护士在床旁、巡视、交接或患者呼叫时获得的异常生命体征和相关症状；模型应输出结构化、专业、可执行的中文护理升级建议。

数据覆盖发热、低氧、低血压、高血压危象、心动过速、心动过缓、低血糖、高血糖、意识改变、胸痛、化疗后发热、产后出血等高风险护理场景，强调及时复测、快速识别危急值、床旁护理处理、规范通知医生和启动急救条件。

## 评判重点

- 是否准确识别异常指标，并能区分轻度异常、潜在恶化和急危重症。
- 是否包含立即复测生命体征、核对测量误差、评估意识/呼吸/循环/疼痛/出血/神经功能等床旁要点。
- 是否给出护理范围内可执行的处理措施，如体位、吸氧准备、心电监护、建立静脉通路、备抢救物品、血糖处理流程、保护安全等。
- 是否明确通知医生、上级护士、快速反应团队、胸痛中心、卒中通道、产科急救等升级条件。
- 是否能用 SBAR 思路记录和交接关键信息，避免遗漏时间、趋势、已采取措施和患者反应。
- 是否保持护理安全边界，不自行诊断、开药、调药、降压、降糖、输液、除颤或延误急救。
- 是否输出合法 JSON，字段完整，语言专业且患者沟通清楚。

## 数据集字段

`dataset.jsonl` 每行一个 JSON 对象，主要字段如下：

- `id`：样例编号。
- `scenario`：固定为“异常生命体征升级建议”。
- `input`：异常生命体征、症状体征、病史背景和现场问题。
- `expected_output`：优秀参考结果，包含：
  - `summary`：当前异常和处置优先级概括。
  - `abnormal_indicators`：异常指标和关键背景。
  - `initial_risk_judgment`：初步风险判断。
  - `immediate_recheck_assessment`：立即复测和评估要点。
  - `nursing_interventions`：护理处理。
  - `escalation_criteria`：通知医生、启动急救或升级条件。
  - `patient_communication`：患者/家属沟通话术。
  - `documentation_handoff`：记录和交接要点。
  - `safety_boundaries`：安全边界。

`prompt.md` 为该场景推荐系统提示词，要求模型只输出 JSON，并包含异常生命体征升级建议的必要字段。
