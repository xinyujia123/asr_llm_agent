# 入院评估测试集

本目录用于评估护理大模型在“入院评估”场景中的结构化抽取、护理风险识别和升级汇报能力。样例覆盖呼吸感染、产后出血、急性卒中、骨折压疮、糖尿病足、消化道出血、老年谵妄、自杀风险、尿毒症高钾、术后出血、急性胰腺炎和急性心衰等不同入院复杂度。

## 文件说明

- `dataset.jsonl`：每行一个测试样例 JSON。
- `prompt.md`：该场景建议使用的系统提示词，要求模型只输出 JSON。
- `README.md`：当前说明文件。

## 数据集字段

`dataset.jsonl` 每条样例至少包含：

- `id`：样例唯一编号。
- `scenario`：固定为 `入院评估`。
- `input`：模拟入院交接、病情描述或护理问诊信息。
- `expected_output`：优秀结果参考，包含评估字段、风险判断、需升级事项和护理优先级。

`expected_output` 主要结构：

- `assessment_fields`：生命体征、主诉、意识/认知、疼痛、过敏史、既往史、用药史、营养/排泄、皮肤压力性损伤、活动能力、伤口/管路以及缺失信息。
- `risk_judgement`：整体危急程度和护理风险，例如跌倒、压力性损伤、误吸、出血、感染/脓毒症、VTE、用药安全、心理安全等。
- `escalation_required`：是否需要升级汇报。
- `escalation_items`：升级紧急程度、对象和依据。
- `nursing_priorities`：本班或入院即刻需要优先执行的护理措施。

## 评判重点

人工评判时建议关注：

1. 是否准确抽取输入中的客观信息，尤其生命体征、过敏史、用药史、伤口/管路、出入量和功能状态。
2. 是否能识别高危或危急情境，并给出合理的整体危急程度。
3. 是否将护理风险具体化，而不是泛泛写“有风险”。
4. 是否明确哪些事项需要立即或尽快升级，并说明依据。
5. 护理优先级是否可执行、与风险匹配，并符合护理安全原则。
6. 是否避免编造输入中没有的信息，同时能列出需要补充评估的关键缺失项。
