# 压疮/跌倒/VTE 风险提醒测试集

本目录用于评估护理大模型在“压疮/跌倒/VTE 风险提醒”场景中的结构化风险识别、证据归因和护理措施生成能力。样例覆盖压力性损伤、跌倒/坠床、VTE 单项风险，以及骨折制动、脑卒中、ICU 镇静、产后、肿瘤、脊髓损伤、抗凝出血风险和医疗器械相关压力损伤等复合风险。

## 文件说明

- `dataset.jsonl`：每行一个测试样例 JSON。
- `prompt.md`：该场景建议使用的系统提示词，要求模型只输出 JSON。
- `README.md`：当前说明文件。

## 数据集字段

`dataset.jsonl` 每条样例至少包含：

- `id`：样例唯一编号。
- `scenario`：固定为 `压疮/跌倒/VTE风险提醒`。
- `input`：模拟护理交接、风险评分、病情变化或床旁观察信息。
- `expected_output`：优秀结果参考，包含风险类别、风险等级、证据、护理措施、复评频率、升级/通知条件、禁忌或注意事项。

`expected_output` 主要结构：

- `risk_assessment`：按压疮、跌倒、VTE 分项列出风险等级和输入证据。
- `overall_priority`：当前护理优先级，例如常规预防、本班重点、尽快处理或立即处理。
- `nursing_measures`：分别给出压力性损伤预防/处理、跌倒预防和 VTE 预防措施。
- `review_frequency`：各风险项目的复评或观察频率。
- `escalation_notification_conditions`：需要通知医生、护士长、伤口造口护士、康复治疗师或药师等的触发条件。
- `contraindications_precautions`：禁忌或注意事项，例如疑似 DVT 禁止按摩患肢、出血风险需评估抗凝安全、约束需遵守制度等。
- `missing_information`：完成风险判断仍需补充的关键信息。

## 评判重点

人工评判时建议关注：

1. 是否准确识别压疮、跌倒、VTE 的单项和复合风险，并能区分高危、极高危、已有损伤或疑似事件。
2. 风险等级是否有输入证据支撑，避免只输出笼统“有风险”。
3. 护理措施是否具体、可执行，并与风险类别匹配。
4. 是否给出合理复评频率，体现病情变化、术后、制动、意识变化等动态场景。
5. 是否明确升级/通知条件和对象，尤其已有压力性损伤、疑似 DVT、跌倒后损伤、出血与抗凝冲突等情况。
6. 是否写出关键禁忌和注意事项，避免危险建议，如疑似 DVT 按摩热敷、未评估即下床、忽视抗凝出血风险等。
